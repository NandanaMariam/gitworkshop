Welcome to Git Basics

What is Git?

Git is a distributed version control system that helps developers collaborate on projects and track changes in source code during software development.

Key Features of Git:

  • Tracks changes to files and directories.
  • Supports collaboration across teams.
  • Allows branching and merging for feature development.

Basic Git Commands:

Initialize a Git repository:
git init
Clone a repository:
git clone <repo_url>
Check the status of changes:
git status
Stage changes:
git add <file>
Commit changes with a message:
git commit -m "message"

Step-by-Step Git Workflow

Here’s a simple guide to help you understand a typical Git workflow:

  1. Initialize a Repository:
    Start a new project by creating a Git repository: git init
  2. Stage Your Changes:
    Add the files you want to track: git add <file_name> Or stage all files at once: git add .
  3. Commit Your Changes:
    Save your changes to the repository: git commit -m "Describe your changes"
  4. Connect to a Remote Repository:
    Link your local repository to a remote one on GitHub: git remote add origin <remote_url>
  5. Push Changes to Remote:
    Upload your changes to the remote repository: git push -u origin main
  6. Pull Updates:
    Keep your local repository up-to-date with the remote: git pull

Repeat these steps as you work on your project. This workflow helps you keep track of changes and collaborate efficiently with others!

  • git init – Initialize a new Git repository.
  • git clone <repo_url> – Clone a repository from a remote server.
  • git status – Check the status of your working directory.
  • git add <file> – Stage changes for commit.
  • git commit -m "message" – Commit changes with a message.
  • git push – Push changes to a remote repository.
  • git pull – Fetch and merge changes from a remote repository.

Why Use Git?

Git is widely used in the software development industry because of its speed, efficiency, and support for collaborative workflows.
